What Mantis does

Not a chat wrapper.
A codebase intelligence engine.

Every feature below is live and works today. No vaporware.

🧠

Persistent Project Memory

After every session, Mantis updates BRAIN.md, appends DECISIONS.log, and records failed approaches in REJECTED.md. The AI never re-suggests what you already rejected.

🕸️

Live Dependency Graph

Run mantis init once. Mantis builds an AST graph of every symbol, import, and caller in your project — and queries it in plain English.

💥

Impact Analysis

Before you touch a function, ask mantis impact <symbol>. Mantis traces every downstream caller and tells you exactly what breaks — before you write a line.

🎯

4-Tier Model Routing

Every query is classified — trivial, code, reasoning, or heavy — and sent to the right model automatically. Fast questions get fast models. Hard questions get the big guns.

🔍

Hallucination Guard

AI output is cross-checked against GROUND_TRUTH.json — a live index of your real function signatures. Invented names get flagged before you ship them.

📐

Architecture Lint

Define layer rules in .mantisrc.yml — e.g. "no handler may import db directly" — and enforce them in CI. Like ESLint, but for your architecture.

🔎

NL Codebase Queries

Ask in plain English: what imports auth?, dead code in billing/, circular deps in api layer. Mantis queries the graph and answers directly.

💰

Token Cost Tracker

Every session ends with a report showing what you'd have paid on GPT-4o, Claude Sonnet, or Opus — and what you actually paid with Mantis: $0.00.

🖼️

Multimodal Input

Drop a screenshot, architecture diagram, or error image with /vision. Works with any vision-capable Ollama model locally or in the cloud.

Get started

Up and running in 3 steps.

1

Install

Run brew install seedhire/tap/mantis or curl -fsSL .../install.sh | sh. One binary, no dependencies.

2

Sign in with GitHub

Mantis opens a browser tab on first run. One click to authorize — no extra account, no billing setup.

3

Index your project

Run mantis init once in your repo to build the dependency graph. Then just run mantis — it knows your codebase.

Project Memory

Picks up where you left off.

After every session, Mantis updates BRAIN.md with what changed, logs your decisions in DECISIONS.log, and records what failed in REJECTED.md. The AI reads all of it on startup — so you never repeat a mistake or re-explain your architecture.

.mantis/
  BRAIN.md  ← rolling project summary
  CONVENTIONS.md  ← architecture rules
  DECISIONS.log  ← timestamped decisions
  REJECTED.md  ← approaches that failed
  graph.db  ← AST dependency graph
.mantisrc.yml  ← CI lint rules
