What Mantis does

Not a chat wrapper.
A codebase intelligence engine.

Every feature below is live and ships today. No vaporware.

Memory & Context
🧠

Persistent Project Memory

After every session, Mantis updates BRAIN.md, appends DECISIONS.log, and records failed approaches in REJECTED.md. The AI reads all of it on startup — zero re-explaining.

🔄

Session Resume

Sessions are saved to .mantis/sessions/. Next run picks up exactly where you left off — same context, same thread. Semantic embeddings surface relevant past turns automatically.

🤝

Handoff Mode

Run mantis handoff to generate a dense HANDOFF.md — current state, key decisions, everything marked "don't touch". Share with your team or your next AI session.

Codebase Intelligence
🕸️

Live Dependency Graph

Run mantis init once. Mantis builds an AST graph of every symbol, import, and caller across Go, TypeScript, and Python — queryable in plain English or visualized with mantis graph.

💥

Impact Analysis

Before you touch a function, mantis impact <symbol> traces every downstream caller, flags untested paths, and tells you exactly what breaks — before you write a line.

⏱️

Temporal Intelligence

Analyze your git history with mantis hotspots, mantis risky, mantis coupling, and mantis intent — find high-churn danger zones, bus factor risks, and co-change patterns.

🔎

NL Codebase Queries

Ask in plain English: what imports auth?, dead code in billing/, circular deps in api layer. Mantis queries the live graph and answers directly — no grepping.

🖥️

TUI Dashboard

Run mantis tui for a full-screen terminal dashboard — live graph stats, session overview, hotspot heatmap, and architecture violations all in one view.

Safety & Reliability
🔍

Hallucination Guard

AI output is checked against a live index of your real function signatures. When an invented symbol is detected, Mantis re-prompts with the correct signatures and returns a fixed answer — automatically.

📐

Architecture Lint

Define layer rules in .mantisrc.yml — "no handler may import db directly" — and enforce them in CI. Like ESLint, but for your architecture boundaries.

🎯

4-Tier Model Routing

Every query is classified — trivial, code, reasoning, or heavy — and routed to the right model automatically. Trivial questions get fast models. Hard ones trigger chain-of-thought reasoning.

Get started

Up and running in 3 steps.

1

Install

Run brew install seedhire/tap/mantis or curl -fsSL .../install.sh | sh. One binary, no dependencies.

2

Sign in with GitHub

Mantis opens a browser tab on first run. One click to authorize — no extra account, no billing setup.

3

Index your project

Run mantis init once in your repo to build the dependency graph. Then just run mantis — it knows your codebase.

Project Memory

Picks up where you left off.

After every session, Mantis updates BRAIN.md with what changed, logs your decisions in DECISIONS.log, and records what failed in REJECTED.md. The AI reads all of it on startup — so you never repeat a mistake, re-explain your architecture, or lose context between sessions.

.mantis/
  BRAIN.md  ← rolling project summary
  CONVENTIONS.md  ← architecture rules
  DECISIONS.log  ← timestamped decisions
  REJECTED.md  ← approaches that failed
  HANDOFF.md  ← team/AI handoff brief
  graph.db  ← AST dependency graph
  sessions/  ← persisted chat history
.mantisrc.yml  ← CI lint rules
