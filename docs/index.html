Platform Overview

Everything Mantis ships today.
Organized by what you need to do.

Use Mantis as an AI assistant, an intelligence layer for your repos, and a guardrail system for safer code changes.

1) Ask, Build, and See (Interactive + Multimodal)
💬

Interactive + One-Shot CLI

Run mantis for full interactive mode, or ask directly with mantis "your question" for quick one-shot answers.

🖼️

Image & Screenshot Input

Analyze UI bugs and diagrams with mantis --image screenshot.png or use /vision inside chat for multimodal debugging workflows.

🧭

Slash Command Workflow

Use focused controls like /file, /brain, /decision, /reject, and /cost without leaving the session.

🧩

25 Built-In Skills + Custom Skills

Mantis ships with 25 built-in specialist skills and also loads any additional markdown skill files you add under .mantis/skills/.

2) Memory That Persists Across Sessions
🧠

Project Brain Files

Mantis continuously updates BRAIN.md, DECISIONS.log, and REJECTED.md so context survives from one day to the next.

✅

Live Ground Truth

GROUND_TRUTH.json stores real function signatures from your codebase and powers automatic hallucination detection + correction.

🔎

Semantic Recall

Past sessions are embedded into embeddings.db so Mantis can retrieve relevant historical decisions when similar problems come back.

3) Codebase Intelligence (Single Repo + Cross Repo)
🕸️

AST Dependency Graph

mantis init builds a graph across Go, TypeScript, and Python for accurate symbol lookups, import tracing, and dependency analysis.

💥

Impact Before Edits

Run mantis impact <symbol> to trace direct + transitive callers and see what change blast radius you are about to create.

🧩

Cross-Repo Workspace Graph

Use mantis workspace init/find/impact/stats to reason about symbol usage and change impact across multiple repositories at once.

4) Risk Detection From History + Runtime
⏱️

Temporal Intelligence

Commands like hotspots, risky, coupling, intent, todos, and spec-gaps identify churn and risk trends in git history.

📈

Runtime Trace Analysis

Ingest OTLP, pprof, or custom traces with mantis trace ingest, then inspect hotpaths, cold, and runtime-weighted impact with trace weight.

🧹

Dead Code + Circular Dependency Checks

Use mantis dead and mantis circular to catch structural issues early, before they become reliability and maintenance debt.

5) Guardrails, Routing, and Cost Visibility
📐

Convention + Architecture Enforcement

Define rules in .mantisrc.yml and run mantis lint to keep boundaries clean in local development and CI pipelines.

🎯

7-Tier Model Router + Vision Tier

Queries route automatically across trivial, fast, code, reason, heavy, max, and vision tiers for better speed/quality balance without manual model switching.

💸

Token Savings Report

Every session includes a cost summary so you can compare against paid models and understand exactly how much money your workflow is saving.

Get started

Up and running in 4 steps.

1

Install

Run brew install seedhire/tap/mantis or curl -fsSL .../install.sh | sh. One binary, no dependencies.

2

Sign in with GitHub

Mantis opens a browser tab on first run. One click to authorize — no extra account, no billing setup.

3

Index your project graph

Run mantis init once inside your repo. This enables impact analysis, symbol search, dead code checks, and smarter context bundling.

4

Start building

Open interactive mode with mantis, ask one-shot questions, or add --image for multimodal debugging workflows.

Project Memory

Persistent memory with verification built in.

Mantis stores human-readable project memory, session context, semantic embeddings, and a live ground-truth signature index. On startup, it reloads this state so answers stay consistent with your actual codebase and previous decisions.
