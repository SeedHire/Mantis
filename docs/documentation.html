Mantis Documentation

v0.7.6 — Free AI coding assistant, local-first, Ollama-powered

Installation

Prerequisites

  • Go 1.21+ — go version
  • Ollama — download from ollama.ai
# Pull at least one model
ollama pull llama3.1           # local fallback
# OR use Ollama Cloud (free tier)
export OLLAMA_API_KEY=your_key

Install Methods

# macOS (Homebrew — recommended)
brew install seedhire/tap/mantis

# macOS / Linux (shell script)
curl -fsSL https://raw.githubusercontent.com/seedhire/mantis/main/install.sh | sh

# Windows (paste into any terminal — cmd, PowerShell, or Windows Terminal)
powershell -c "irm https://raw.githubusercontent.com/seedhire/mantis/main/install.ps1 | iex"

# Any platform (requires Go 1.21+)
go install github.com/seedhire/mantis/cmd/mantis@latest

# Build from source (any platform)
git clone https://github.com/seedhire/mantis
cd Mantis && make build && make install
mantis --version   # mantis v0.7.6 (darwin/arm64)

First Run

# With GitHub auth (enables cloud features)
mantis

# Without GitHub auth (fully local, no internet required)
mantis --offline

First interactive run triggers a one-time setup: GitHub login (optional) and Ollama connection check. If auth fails, use --offline to skip entirely.

Tip: The --offline flag is perfect for air-gapped environments or when you just want local Ollama models.

Index Your Project

Run mantis init once in any project. This parses every file, builds a SQLite dependency graph (.mantis/graph.db), and creates project memory files.

cd your-project/

mantis init              # auto-detect language
mantis init --lang go    # specify primary language
mantis init --lang ts --watch   # index + watch for changes
FlagDescription
--langPrimary language: go, ts, py, rust, java
--watchStart file watcher immediately after init
Note: .mantis/ is auto-added to .gitignore. The graph DB never leaves your machine.

Interactive REPL

The main mantis command opens an interactive session with 22 slash commands.

mantis                    # interactive REPL
mantis "why is checkout timing out?"    # one-shot question
mantis --image screenshot.png "what is wrong?"   # multimodal
mantis --plan "build a REST API"        # plan-first mode
mantis --continue                       # resume last session
mantis --model max                      # force model tier

How Each Message Is Processed

  1. Router classifies intent into 1 of 7 tiers, selects the best model
  2. Graph context auto-injects related files from the dependency graph
  3. Memory retrieves relevant past decisions via hybrid BM25+cosine search
  4. Model generates response (streamed live)
  5. Verify checks against GROUND_TRUTH.json (hallucination detection)
  6. Verify checks against CONVENTIONS.md (architecture convention gate)
  7. Diff preview shown before any file is written

Diff Preview

Every file change shows a colored diff with an approval prompt before anything is written to disk.

┌─ Preview: internal/auth/service.go (existing) ─────────┐
│   23   - func (s *Service) Login(email string) error {  │
│   23   + func (s *Service) Login(ctx context.Context,   │
│   24   +     email string) (*User, error) {             │
└─────────────────────────────────────────────────────────┘
Write this file? [Y/n]

Y or Enter to write. n to skip. Changes are never applied without explicit confirmation.

Code Generation Pipeline

For complex build requests, a multi-stage pipeline activates automatically.

mantis "build a REST API with express, typescript, JWT auth, and postgres"
mantis --plan "build a todo app with react and node"   # review plan first

Stages

StageWhat Happens
PLANReasoning model decomposes into 6-10 tasks with file assignments
CODETasks execute with live progress. Tasks 0-1 sequential (setup + types), rest parallel. Build verification + up to 3 fix retries per task.
TESTSTest files generated per task
VERIFYTestLoop runs tests and iteratively fixes failures (up to 3 iterations)
Sealed Types: After sequential tasks generate shared types, a SEALED TYPES manifest is extracted and injected into parallel workers to prevent type redefinition.

Agent System

For high-impact changes (4+ files across 2+ packages), the multi-agent orchestrator activates automatically.

mantis "add OAuth2 support to the auth package, update the middleware, and add integration tests"

Workers run in parallel, each with access to 7 tools:

ToolWhat It Does
read_fileRead any file in the project
write_fileWrite a new file
edit_fileTargeted SEARCH/REPLACE edit on existing file
run_bashRun allowed shell commands (go/npm/cargo/pytest/git)
search_codebaseSemantic search across all files
find_symbolFind a symbol in the dependency graph
run_testsRun tests and get parsed results

Iterative Test Loop

Mantis can automatically run tests, parse failures, read relevant source code, and prompt the model to fix issues — iteratively.

# From the REPL
/test                     # run all tests
/test ./internal/auth/... # run specific package

# Or just describe the problem
> fix the failing tests in auth package

Supports Go, Node/Jest, Python/pytest, Rust, with automatic runner detection. Stuck detection stops if the same failures persist after a fix attempt.

mantis find

Find all usages of a symbol across the codebase.

mantis find "UserService" --format tree
mantis find "processPayment" --format json

mantis dead

Detect exported symbols with zero references.

mantis dead --ignore "**/*.test.ts,**/*.d.ts"

mantis circular

Find circular import chains with optional HTML visualization.

mantis circular --html --fix

mantis graph

Generate an interactive D3 dependency graph HTML file.

mantis graph --module src/auth --depth 3

mantis context

Bundle all files related to a symbol into a structured Markdown block optimized for LLM input.

mantis context "AuthMiddleware" --tokens 16000 | pbcopy

Files with highest churn. Splits into Refactor Candidates (single owner, many commits) and Watch List (multi-owner).

Files that change together (hidden structural dependencies).

Conventions Enforcement

Write your architecture rules in .mantis/CONVENTIONS.md. Every AI response is checked against these rules. If a response violates a convention, the AI is re-prompted automatically until it complies.

# .mantis/CONVENTIONS.md

## Imports
- Never import from `internal/legacy/`
- Database access only through repository interfaces

## Naming
- All service constructors must be named `NewXxxService`
- HTTP handlers must accept `(w http.ResponseWriter, r *http.Request)`

## Architecture
- No circular dependencies between packages
- Business logic never in HTTP handlers

Ground Truth Guard

Mantis builds GROUND_TRUTH.json from the live AST — a snapshot of every exported function signature in the codebase. When the AI references a function, the response is checked against ground truth. If the signature doesn't match, the response is flagged and re-prompted.

Result: The AI cannot hallucinate function signatures that don't exist in your codebase.

LSP Server

Mantis provides an LSP server for editor integration (VS Code, Neovim, etc.).

mantis lsp

Supported capabilities: hover (function docs + graph context), go-to-definition, document symbols, diagnostics (lint violations), code lens (impact analysis).

MCP Server

Mantis exposes an MCP (Model Context Protocol) server so other AI tools can access its graph intelligence.

mantis mcp

Exposes tools: impact_analysis, find_symbol, dead_code, hotspots, context_bundle.

CI Integration

Add Mantis to your CI pipeline for architecture rule enforcement.

# GitHub Actions example
- name: Architecture lint
  run: |
    mantis init --lang go
    mantis lint --strict --ci

The --ci flag exits with code 1 on any violation, blocking the PR merge.

Architecture Lint

Configure rules in .mantisrc.yml at the project root.

# .mantisrc.yml
version: 1
rules:
  - name: no-circular-dependencies
    type: built_in
    severity: error

  - name: max-file-dependencies
    type: built_in
    severity: warning
    config:
      max: 15

  - name: no-direct-db-in-handlers
    type: custom
    pattern: "handlers/**"
    forbidden_imports:
      - "database/sql"
      - "internal/db"
    severity: error
    message: "Handlers must not import database packages directly"
mantis lint              # check rules
mantis lint --strict --ci   # CI gate (exit 1 on violation)
